# VocabFiction 产品分析
> 一个让对好故事的喜爱转化成英语学习动力的日常词汇积累工具。
>

## 功能描述
用户上传自己喜欢的英文小说（或使用系统内置的作品），选择或上传想要学习的词表。App 将小说内容重写为对话体（快点阅读），逐句点击阅读——像刷短视频一样往下翻。目标词汇在阅读中自然出现：第一次带中文释义建立连接，后续再出现时不带释义，用户可点击查询，连续多次不查则视为已学会。遇到难以嵌入剧情的目标词时，每 N 集会插入一集 AI 生成的番外来覆盖。用户不需要在阅读之外做任何事——没有测验、没有打卡、没有待复习列表。读完一集后看到这份积累（本集 X 个词、累积 Y 个词），然后被故事拉着去读下一集。

---

## 目标用户
一个人，同时满足两个条件：

1. **喜欢故事** — 碎片时间里，比起打开背单词 App，更愿意读小说、刷剧、看动漫、看漫画等等
2. **想提升英语但坚持不了传统工具** — 有意愿提升英语水平和词汇量，但百词斩、Duolingo 这类产品坚持不下来

---

## 痛点
### 痛点一：长期英语学习难以启动和坚持
目标用户并非不想学，而是找不到能持续的方式。

+ **启动困难**：打开背词 App 需要调动意志力。"该背单词了"触发的不是行动，而是逃避反射
+ **持续困难**：传统工具靠外部压力（打卡、通知、考试）维持用户，压力消失行为立即终止。用户反复经历"下载→坚持→放弃→内疚→再下载"的循环

用户打开这个产品的动机是两根绳子拧在一起的：**学英语的意愿 + 看故事的意愿**。两者缺一不可。

### 痛点二：传统方法缺乏情景，背了调不出来
背词软件把词汇从使用环境中剥离——用户记住了中文释义，却不知道在真实场景中怎么用。

+ 阅读时能识别（"我背过这个词"）
+ 写作/说话时调不出来（词库没有情境锚点）

---

## 产品能力边界
只负责**输入端**——帮助用户积累有情景锚点的词汇库存。"有词可调"是这个产品的终点。确保用户在需要调用的那一刻，词是存在的、有情境路径可到达的。但产品自身不承诺完成从输入到输出的全链条。

+ **Why:** 识别（阅读中认出）和回忆（写作/说话时主动调用）是两个不同级别的能力。阅读训练无法直接产出写作能力。该产品做的是识别端的积累和留存。
+ **How to apply:** 所有产品设计不应追求"测得出"的效果（填空、翻译、拼写），而应追求"用户每天自然接触英语并累积词汇"。

---

## 设计原则（按优先级排序）
前三条保障用户能持续阅读，后两条保障阅读中确实学到东西。

### 1. 不给用户制造压力
+ 断开了再打开，没有任何"你欠了什么"的痕迹。无红点、无待复习计数、无缺席提醒
+ 成长数据只用于让用户自己发现，不主动推送、不用于比较、不制造焦虑
+ 不做排行榜、不做用户比较、不出"你的进度落后了"类信息
+ 所有统计（已读集数、词数、遇见的新词数）的措辞是盘点资产，不是催缴账单

### 2. 使用门槛降到最低
+ 打开即读，不需要导航、选择、加载
+ 阅读中不被打断——不弹窗、不提问、不要求阅读之外的操作
+ 释义直接嵌入文本，不需要离开对话流查词
+ 唯一需要的操作：点击屏幕看下一句
+ 不要求读完一集，读半句也可以关掉

### 3. 读完即正反馈
+ 读完一集，"我做到了"的感觉是用户带走的东西
+ 集末词汇面板是呈现而非考验——"本集你遇到了 X 个词"，不是"请学习以下单词"
+ 面板可同时展示累积统计：已读 X 集、X 词、遇见 X 个新词、阅读 X 分钟

### 4. 读就是学
+ 用户唯一需要做的事是阅读，词汇习得是阅读的自然伴随物
+ 不存在独立于阅读的"学习模式"。没有测验、没有专项练习、没有需要切换的任务
+ 读就是学——这是产品区别于所有传统学习工具的本质特征

### 5. 重复对目标词的回忆
+ 目标词第一次出现时带中文释义，建立初始连接
+ 后续跨集复现时不再展示释义，用户可点击查询中文
+ 连续多次未查询中文会通过 FSRS 计算延长该词的复习间隔（due），使其在调度中自然退场
+ 复现由系统后台自动安排，用户不需要知道"复习"的存在
+ 既不为了教学扭曲剧情，也不为了流畅丢弃词汇目标

---

## 核心循环
```plain
读故事 → 遇到目标词 → 看到释义 → 继续读 → 再遇见 → 回忆起来释义
→ 继续读 → 不断遇见 → 不断回忆 → 记住单词 → 继续读
```

+ 用户唯一主动做的事只有一项：**读故事**
+ "遇到目标词""看到释义""回忆起来""记住单词"都是读的过程中自然发生的事，不是独立的任务
+ 产品在后台保证目标词的跨集复现，让"不断遇见→不断回忆"持续运转

## 功能清单
### 内容供给层
+ 提供可连续阅读的英文故事（系统内置 + 用户上传）
+ 目标词嵌入故事中
+ 番外集：每 N 集插入一集 AI 全量生成的番外，覆盖难以嵌入主线剧情的目标词

### 基础层
+ 分集
+ 逐句阅读，对话式为主，提供常规阅读器模式作为备选
+ 目标词第一次出现时展示中文释义
+ 后续出现时不展示释义，用户可点击查询中文
+ 连续 M 次出现用户均未查询，视为已学会

### 核心层
+ 词汇复现机制：安排目标词在后续剧集中跨集复现
+ 番外集生成：AI 根据待覆盖词表生成独立短篇故事

### 反馈层
+ 单集结束后展示本集词汇
+ 展示累积阅读数据
+ 提供多部作品

## 内容生成方向分析
### 输入方向
**方向一：经典材料做翻译/改写/嵌入**

+ 优：故事质量有保障，来源稳定
+ 劣：词汇嵌入受限，只能在原文基础上做替换和改写，替换过多会破坏原文味道

**方向二：用户提供材料，调用大模型处理**

+ 优：个性化最强，用户对故事有情感连接，认知负荷低
+ 劣：词汇嵌入受限，原文词汇分布不为你控制；用户需要先做上传动作

**方向三：大模型根据词表直接创作**

+ 优：词汇嵌入完全可控，可以根据任何词表定制
+ 劣：故事质量没有保证，大模型创作的文本容易被用户感知为"AI味儿"

### 输出方向
**输出一：普通阅读器**

段落叙事，类似传统电子书阅读。

+ 优：适用面广，所有内容类型都能呈现，不挑源材料
+ 优：实现成本低，不需要重构内容结构
+ 劣：和市面上的英文阅读器差异不大，没有产品特色
+ 劣：阅读难度较高，用户不容易坚持

**输出二：对话式小说**

聊天体，逐句点击阅读。

+ 优：点击逐句展开，像刷短视频一样有「下一条」机制，容易形成阅读习惯
+ 优：句子短小简单，用户认知压力低，更容易读下去
+ 优：句子短，适合碎片时间阅读
+ 劣：对源材料要求高，不是所有内容都适合对话体改写
+ 劣：不适合的内容强行改对话体，阅读体验会很奇怪

---

## 最终选择
**输入**：方向二（用户提供材料 + 系统内置），配合方向三（番外集）。

+ 主力是用户上传自己的小说，系统内置部分作品作为起步内容
+ 上传内容覆盖不了的目标词通过番外集（方向三）补充

**输出**：输出二（对话式小说）为主，输出一（常规阅读器）为备选。

+ 默认以对话体呈现，逐句点击阅读
+ 用户可随时切换普通段落阅读模式，应对不适合对话体的内容

## 竞品分析
### 一、纯阅读器类（双语对照 + 有声）
**Beelinguapp**

+ 核心：双语并行阅读（原文+母语逐句对照），卡拉OK式音频跟读
+ 内容：童话、短篇故事、新闻、科普，部分AI生成
+ 弱点：没有真正的间隔重复系统，闪卡很基础，内容质量参差不齐
+ 收费：年费 ~$40

**LingQ**

+ 核心：海量内容库 + 用户可导入任何内容（YouTube、Netflix、电子书），点击查词自动收集到词库，SRS间隔复习
+ 2025年加了AI简化课程、AI聊天机器人
+ 弱点：对初学者不友好，UI老旧，价格涨了之后争议大
+ 收费：Premium ~$ 8.99/月，Premium Plus ~ $40/月

### 二、"阅读+学习" 混合型
**薄荷阅读（百词斩旗下）**

+ 核心：名著分级阅读（傲慢与偏见、福尔摩斯等），原文+翻译+配音
+ 2025年10月上线了"英文互动小说"功能：AI生成画面、剧情分支选择、多支线重玩
+ 收费：19.9元/部，单篇付费
+ 评价：沉浸感目前还比较简单，比较像"有插图的阅读器+选择"

**扇贝阅读**

+ 核心：分级阅读 + 短文 + 查词 + 生词本
+ 内容：短文为主，也有名著，但比不上薄荷的书库规模
+ 特点：和扇贝背单词打通词库

### 三、对话/AI角色互动型
**Storist**

+ 核心：读短篇故事 → 与 AI 角色对话（Homer Simpson 等）→ 闪卡记忆
+ Read → Speak → Memorize

**Lingomate**

+ 核心：互动小说 + 雅思词汇嵌入 + 语音播放，剧情选择驱动
+ 偏游戏化，章节结尾归纳新词

**StoryPlay AI**

+ 核心：AI 生成角色扮演故事，可选择难度 A1-C2
+ 偏角色扮演游戏风

### 四、背单词 App
**百词斩**

+ 核心：图片联想记忆，词库覆盖广（四六级、考研、雅思等）
+ 特点：图文结合，部分内容有声，社区功能
+ 弱点：图片关联强但脱离语境，容易"记住了图却没记住词"

**墨墨背单词**

+ 核心：艾宾浩斯遗忘曲线 + 真实例句
+ 特点：极简、无多余功能，专注"背"的效率
+ 弱项：纯工具，没有语境沉浸感，枯燥但高效

**不背单词**

+ 核心：真实语境例句（影视、新闻、小说原文摘录）
+ 特点：在句子中背单词，比传统闪卡有语境感
+ 弱点：例句来源随机，不成体系，缺乏连续语境

### 五、综合语言学习 App
**Duolingo（多邻国）**

+ 核心：游戏化闯关 + 听说读写全覆盖，支持 40+ 语言
+ 2025年方向：AI视频通话、角色扮演、LinkedIn分数认证、国际象棋课
+ 中国是增长最快市场，DAU 4770万，付费转化率 8.7%
+ 弱点：娱乐性强但深度有限，学到中高级后效率偏低；答错/对都消耗能量，变相推付费

**流利说**

+ 核心：AI语音评分 + 自适应课程（懂你英语A+），主打口语
+ 内容：8级体系对应CEFR/雅思，配音课、真人语伴互动
+ 收费：年卡 ~¥986，有打卡返现机制
+ 弱点：课程导向，学习任务重，需要用户主动投入意志力

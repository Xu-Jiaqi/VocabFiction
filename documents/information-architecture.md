# VocabFiction 信息架构

## 页面地图

```
App
├── 书架（首页）
│   ├── 作品卡片列表
│   ├── 点击内置作品 → 阅读界面
│   ├── 点击用户上传作品 → 无反应
│   ├── 长按作品 → 作品管理页
│   └── 底部 "+" → 上传小说 / 上传词表
│
├── 阅读界面（主场景）
│   ├── 状态栏（作品名 + 集数 + 进度条）
│   ├── 消息列表（可滚动）
│   ├── 底部点击提示
│   ├── 角落设置入口
│   ├── 顶部返回 → 书架
│   ├── 集末词汇面板（半屏浮层）
│   └── 词典面板（顶部浮层）
│
├── 上传页
│   └── 返回 → 书架
│
├── 作品管理页
│   ├── 修改作品名称
│   ├── 更换绑定词表
│   ├── 用户上传作品：删除小说
│   └── 内置作品：不可删除
│
├── 设置（浮层面板）
│   ├── 字体大小、阅读模式
│   └── API 设置入口 → API 设置页
│
└── API 设置页
    └── 返回 → 来源页
```

| 从 | 到 | 触发 | 方式 |

### 1. 书架（首页）

```
┌─────────────────────────────────┐
│  VocabFiction              [⚙] │  ← 原生 Stack header
│  ┌─────────────────────────┐    │
│  │ 📖 傲慢与偏见            │    │  ← 作品卡片
│  │ 📖 傲慢与偏见            │    │  ← 作品卡片
│  │    NJU词汇表AB类汇总       │    │
│  ├─────────────────────────┤    │
│  │ 📖 小王子                │    │
│  │    NJU词汇表AB类汇总       │    │
│  ├─────────────────────────┤    │
│  │ 📖 我的上传作品          │    │
│  │    我的四级词表            │    │
│  └─────────────────────────┘    │
│                                 │
│                          [+]    │  ← 底部添加入口
└─────────────────────────────────┘
```

**设计要点：**
- 卡片背景 `#FAF8F3`，与主背景同色，用底部细线（`#E5E0D5`）分隔
- 卡片内容：作品名（衬线体，15px，`#2C2416`）+ 绑定词表名（sans-serif，12px，`#A09C94`）
- 顶部使用原生 Stack header，标题 "VocabFiction"；右侧设置图标入口
- 卡片顺序：最近阅读的排在最上面
- 内置作品：点击进入 reader；长按进入管理页，可更换绑定词表
- 用户上传作品：点击无反应；长按进入管理页，可更换词表或删除
- 不使用封面缩略图，纯文字排版（保持简洁）
- 空书架状态：居中显示"选择一部作品开始阅读" + "上传你的小说 →"
- "+" 按钮：极简，仅文字或细线图标

#### 3.2 上传小说

```
┌─────────────────────────────────┐
│  ← 上传小说                      │  顶部导航
├─────────────────────────────────┤
│  当前词表：NJU AB类（1200 词）    │  可点击“更换”进入上传词表
│                                 │
│  小说文件                        │
│  ┌─────────────────────────┐    │
│  │ 选择 .txt 文件           │    │  虚线边框区域
│  └─────────────────────────┘    │
│                                 │
│  作品名称                        │
│  ┌─────────────────────────┐    │
│  │ 输入名称（选填）         │    │  输入框
│  └─────────────────────────┘    │
│                                 │
│  小说内容                        │
│  ┌─────────────────────────┐    │
│  │ 或直接粘贴小说文字       │    │
│  └─────────────────────────┘    │
│                                 │
│  ┌─────────────────────────┐    │
│  │        保存到书架 →      │    │  按钮
│  └─────────────────────────┘    │
└─────────────────────────────────┘
```

**设计要点：**
- 上传区域用虚线边框 + 浅灰背景展示，不用实心卡片
- 输入框样式简洁：底部细线，无边框
- 按钮使用文字 + 箭头，低调不抢眼（"继续阅读"同风格）
- 上传词表页负责创建 `wordListId`，保存词表内容，并更新“当前词表”
- 上传小说页读取当前词表，保存 UTF-8 原文和绑定关系，并在 App 内批量生成 episode JSON
- 上传小说必填：小说文本（文件或粘贴）。作品名称可选，不填则从文件名提取
- 生成成功后用户上传作品可直接进入 reader；未生成完成时点击进入管理页继续生成

**上传后流程：**
```
用户先上传/粘贴词表 → 保存为当前词表（生成 wordListId）
  → 上传/粘贴小说
  → 点击"生成并加入书架"
  → 本地保存 documentDirectory/novels/<workId>/plain.txt 和 meta.json
  → 写入 works 表（total_eps = 0），并记录 works.word_list_id
  → App 内运行生成 pipeline，持续写 generation-checkpoint.json
  → 生成 episodes/、chapters.json、arc-plan.json、vocabulary.json
  → 更新 works.total_eps 并进入 reader
```

**异常处理：**
- 文件格式不支持 → 明确提示支持 .txt
- 文件过大 → 提示大小限制
- 保存失败 → 页面内提示失败，可重试
- 词表为空或格式不正确 → 提示"词表为空，请检查文件格式（每行一个单词）"

### 4. 作品管理页

从书架长按任意作品进入。

```
┌─────────────────────────────────┐
│  ← 管理作品                      │
├─────────────────────────────────┤
│  作品                            │
│  [败犬女主太多了！____________]   │
│  内置小说                         │
│                                 │
│  绑定词表                         │
│  当前：NJU词汇表AB类汇总           │
│  ┌─────────────────────────┐    │
│  │ NJU词汇表AB类汇总     ✓  │    │
│  ├─────────────────────────┤    │
│  │ 我的四级词表             │    │
│  └─────────────────────────┘    │
│                                 │
│  [删除小说]（仅用户上传作品显示）   │
└─────────────────────────────────┘
```

**交互规则：**
- 所有作品都可修改作品名称；输入框失焦或回车时保存到 `works.title`。
- 所有作品都可更换绑定词表，包括内置小说。
- 内置小说默认绑定 `builtin-nju-ab`；管理页只允许更换词表，不允许删除。
- 用户上传小说可删除；删除会清理 `documentDirectory/novels/<workId>/` 和 `works` 表记录。
- 更换词表只更新 `works.word_list_id`，不改写小说原文。

## 状态流转

### 阅读位置状态

```
未开始 → 阅读中 → 已读完（可重读）
  │        │
  │        └── 中断（下次打开从此处继续）
  └── 点击开始 → 阅读中
```

### 词汇状态（单用户-(词,释义)对维度）

追踪单位是（word, definition）对。同一个词不同义项独立追踪。

```
未出现 → 首次出现（is_new: true，释义可见）
       → 复习出现（is_new: false，加粗，可点击）
       → 连续 3 次未查询 → 已学会（普通文字）
                           ↕ 用户可手动撤销
```

例：`bank=河岸` 和 `bank=银行` 是两个独立的学习对象，各自有自己的 `is_new` 生命周期和 M 值计数。

### 上传作品状态

```
未上传 → 已保存原文（书架可见，total_eps = 0）
       → 生成中/失败（管理页可继续生成）
       → 已生成分集（点击进入 reader）
       → 长按管理 → 更换词表 / 删除小说（移除本地目录和 works 记录）
```

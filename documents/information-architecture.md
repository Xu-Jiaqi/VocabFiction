# VocabFiction 信息架构

## 页面地图

```
App
├── 书架（首页）
│   ├── 作品卡片列表
│   ├── 右上角 "+" → 上传页
│   └── API 设置入口（未配置时提示）
│
├── 阅读界面（主场景）
│   ├── 状态栏（作品名 + 集数 + 进度条）
│   ├── 消息列表（可滚动）
│   ├── 底部点击提示
│   ├── 角落设置入口
│   ├── 顶部返回 → 书架
│   ├── 集末词汇面板（半屏浮层）
│   └── 词典面板（顶部浮层）
│
├── 上传页
│   └── 返回 → 书架
│
├── 设置（浮层面板）
│   ├── 字体大小、阅读模式
│   └── API 设置入口 → API 设置页
│
└── API 设置页
    └── 返回 → 来源页
```

## 页面关系与导航

```
                    ┌──────────┐
                    │   书架    │  ← App 入口 / 首页
                    │ (Home)   │
                    └────┬─────┘
                         │
          ┌──────────────┼──────────────┐
          │ 点击作品      │ 点击 "+"     │
          ▼              ▼              │
   ┌──────────┐   ┌──────────┐         │
   │ 阅读界面  │   │  上传页   │         │
   │ (Reader) │   │ (Upload) │         │
   └────┬─────┘   └────┬─────┘         │
        │              │               │
   ┌────┴─────┐        │               │
   │ 集末面板  │        │               │
   │ 词典面板  │        │               │
   │ 设置     │  → API 设置页 ←────────┘
   └──────────┘    (可从书架/设置/上传进入)

   返回 ←────────────────────────────────┘
```

## 导航规则

| 从 | 到 | 触发 | 方式 |
|---|---|---|---|
| 书架 | 阅读界面 | 点击作品 | push（从右进入） |
| 书架 | 上传页 | 点击 "+" | push（从右进入） |
| 书架 | API 设置页 | 未配置 API 时点击提示 | push |
| 阅读界面 | 书架 | 点击返回 / 系统返回键 | pop（从左退出） |
| 阅读界面 | 设置 | 点击角落设置图标 | 浮层从底部弹出 |
| 设置 | API 设置页 | 点击"API 设置"入口 | push |
| 上传页 | 书架 | 点击返回 / 上传完成 | pop |
| 上传页 | API 设置页 | 未配置 API 时点击"去配置" | push |
| 阅读界面 | 集末面板 | 读完最后一条消息 | 从底部滑入（不导航） |
| 阅读界面 | 词典面板 | 点击词汇 → 点击小卡片 | 从顶部滑入（不导航） |
| API 设置页 | 返回 | 点击返回 | pop（回到来源页） |

## 各页面详细设计

### 1. 书架（首页）

```
┌─────────────────────────────────┐
│  VocabFiction                   │  ← 应用名，温灰小字
│                                 │
│  ┌─────────────────────────┐    │
│  │ 📖 傲慢与偏见            │    │  ← 作品卡片
│  │    已读 Ep.3 / 共 12 集  │    │
│  │    3 天前阅读             │    │
│  ├─────────────────────────┤    │
│  │ 📖 小王子                │    │
│  │    未开始 / 共 8 集      │    │
│  ├─────────────────────────┤    │
│  │ 📖 我的上传作品          │    │
│  │    已读 Ep.1 / 生成中…   │    │
│  └─────────────────────────┘    │
│                                 │
│                          [+]    │  ← 右上角上传入口
└─────────────────────────────────┘
```

**设计要点：**
- 卡片背景 `#FAF8F3`，与主背景同色，用底部细线（`#E5E0D5`）分隔
- 卡片内容：作品名（衬线体，15px，`#2C2416`）+ 阅读状态（sans-serif，12px，`#A09C94`）
- 卡片顺序：最近阅读的排在最上面
- 不使用封面缩略图，纯文字排版（保持简洁）
- 空书架状态：居中显示"选择一部作品开始阅读" + "上传你的小说 →"
- "+" 按钮：极简，仅文字或细线图标

### 3. 上传页

```
┌─────────────────────────────────┐
│  ← 上传作品                      │  顶部导航
├─────────────────────────────────┤
│                                 │
│  小说文件                        │
│  ┌─────────────────────────┐    │
│  │ 点击上传 .txt 文件       │    │  虚线边框区域
│  │ 或直接粘贴文字           │    │
│  └─────────────────────────┘    │
│                                 │
│  作品名称                        │
│  ┌─────────────────────────┐    │
│  │ 输入名称（选填）         │    │  输入框
│  └─────────────────────────┘    │
│                                 │
│  词表文件                        │
│  ┌─────────────────────────┐    │
│  │ 点击上传 .txt 词表       │    │  虚线边框区域
│  │ 每行一个单词             │    │
│  └─────────────────────────┘    │
│                                 │
│  ┌─────────────────────────┐    │
│  │        开始生成          │    │  按钮
│  └─────────────────────────┘    │
│                                 │
└─────────────────────────────────┘
```

**设计要点：**
- 上传区域用虚线边框 + 浅灰背景展示，不用实心卡片
- 输入框样式简洁：底部细线，无边框
- "开始生成"按钮：文字 + 箭头，低调不抢眼（"继续阅读"同风格）
- 必填项：小说文件（文本或粘贴）+ 词表文件。作品名称可选，不填则从文件名提取
- 上传后进入等待状态，提示"作品正在处理中"，不强制用户等待

**上传后流程：**
```
用户点击"开始生成"
  → 调用用户配置的 API 逐集生成 JSON
  → 返回书架
  → 作品卡片显示"生成中…"状态
  → 生成完成后本地存储，用户可点击进入阅读
  → 失败时卡片显示"生成失败，点击重试"
```

**异常处理：**
- 文件格式不支持 → 明确提示支持 .txt
- 文件过大 → 提示大小限制
- 生成失败 → 作品卡片显示"生成失败，点击重试"
- 词表为空或格式不正确 → 提示"词表为空，请检查文件格式（每行一个单词）"

```
│  └─────────────────────────┘    │

│                                 │
│  API Key                        │
│  ┌─────────────────────────┐    │
│  │ ********************     │    │  安全输入框，密文显示
│  └─────────────────────────┘    │
│  说明：密钥仅存储在本地设备      │
│                                 │
│  模型名称                       │
│  ┌─────────────────────────┐    │
│  │ deepseek-v4-pro          │    │  输入框，预填默认值
│  └─────────────────────────┘    │
│                                 │
│  ┌─────────────────────────┐    │
│  │        测试连接          │    │  按钮
│  └─────────────────────────┘    │
│  测试结果：连接成功 ✓           │  状态反馈（成功/失败）
│                                 │
└─────────────────────────────────┘
```

**设计要点：**
- 输入框底部细线样式，与上传页一致
- "API Key"输入框使用安全输入模式，输入时显示为密文圆点
- 提供显示/隐藏切换（眼睛图标）
- 所有输入即时自动保存到本地安全存储
- "测试连接"：发送一条简短请求，显示结果（成功绿色 ✓ / 失败红色 ✗ + 原因）
- 失败提示区分：网络错误 / 认证失败（401/403） / 超时
- API 地址输入框旁可提供常用提供商快捷切换（OpenAI / Anthropic / 自定义）

**默认配置（内置）：**
| 提供商 | API 地址 | 默认模型 |
|--------|---------|---------|
| DeepSeek | `https://api.deepseek.com` | `deepseek-v4-pro` |

> 用户可修改为任何 OpenAI 兼容接口的地址和模型。

**安全说明：**
- API Key 存储在 iOS Keychain / Android EncryptedSharedPreferences
- 仅向用户配置的地址发送请求
- App 自身不收集、不上传 Key 到任何第三方

## 状态流转

### 阅读位置状态

```
未开始 → 阅读中 → 已读完（可重读）
  │        │
  │        └── 中断（下次打开从此处继续）
  └── 点击开始 → 阅读中
```

### 词汇状态（单用户-(词,释义)对维度）

追踪单位是（word, definition）对。同一个词不同义项独立追踪。

```
未出现 → 首次出现（is_new: true，释义可见）
       → 复习出现（is_new: false，加粗，可点击）
       → 连续 3 次未查询 → 已学会（普通文字）
                           ↕ 用户可手动撤销
```

例：`bank=河岸` 和 `bank=银行` 是两个独立的学习对象，各自有自己的 `is_new` 生命周期和 M 值计数。

### 上传作品状态

```
生成中 → 可阅读（生成成功）
       → 生成失败（可重试）
```

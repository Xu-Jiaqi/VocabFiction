# VocabFiction 用户故事

## US-1: 首次使用 —— 从内置作品开始阅读

**作为**一个想学英语但坚持不了背词 App 的用户，
**我想要**打开 App 后立刻就能开始读故事，
**以便**不需要做任何选择、设置、上传就能体验产品。

### 场景

1. 用户首次打开 App
2. 看到系统推荐的内置作品（如名著改编），附封面、简介、目标词数
3. 点击一部作品，直接进入阅读界面，从 Ep.1 第 1 条消息开始
4. 无需注册、无需选择词表、无需上传任何内容

### 验收要点

- 内置至少 1 部完整作品可供直接阅读
- 首次打开无任何弹窗、引导页、注册要求
- 一进入就是阅读界面，顶部显示作品名和 Ep.1

---

## US-2: 逐句阅读

**作为**一个碎片时间阅读的用户，
**我想要**每次点击屏幕只出现一条消息，
**以便**我可以随时停下，阅读压力小，像刷短视频一样自然。

### 场景

1. 进入阅读界面，看到上一条历史消息（如有）
2. 点击屏幕底部区域 → 下一条消息以动画（slideUp）出现
3. 叙述文字居中灰字淡入
4. 对话以气泡形式出现：
   - 左气泡（他人）：头像在左，角色名在上，气泡内容以 slideUp 动画出现
   - 右气泡（主角）：头像在右，角色名右对齐在上，气泡内容以 slideUp 动画出现
5. 重复点击，读完本集所有消息
6. 最后一条消息出现后，弹出集末词汇面板

### 验收要点

- 唯一操作是点击屏幕
- 历史消息保留在屏幕上，可向上滚动回看

---

## US-3: 遇到生词（首次出现）

**作为**一个词汇量有限的英语学习者，
**我想要**在阅读中看到生词时能直接看到中文释义，
**以便**不用离开阅读流去查词典，不打断阅读节奏。

### 场景

1. 阅读到包含目标生词的句子
2. 该词以**加粗**形式展示，后面紧跟灰色括号释义
   - 示例：**invisible**（隐形的）
3. 用户继续点击阅读，无需任何额外操作

### 验收要点

- 释义颜色为温灰色（`#A09C94`），不喧宾夺主
- 释义只在词后紧跟出现，不弹窗、不浮层
- 只对 `is_new: true` 的词汇展示释义

---

## US-4: 复习词（非首次出现）

**作为**一个已见过某个词但还没记住的用户，
**我想要**在再次遇到时能点击查词，
**以便**不记得的时候能快速确认释义，但不需要每次都看到释义干扰阅读。

### 场景

1. 阅读到已出现过的目标词
2. 该词仅**加粗**展示，无释义
3. 用户记得这个词 → 不需要任何操作，继续读
4. 用户不记得 → 点击加粗词 → 弹出第一级小卡片（词 + 释义）
5. 小卡片浮在词上方，不遮挡当前句子
6. 点击卡片外区域 → 卡片消失

### 验收要点

- `is_new: false` 的词仅加粗，不显示释义
- 小卡片动画为原地淡入 0.15s
- 小卡片圆角 8px，背景 `#F5F2EB`，极淡阴影
- 点击卡片外即可关闭

---

## US-5: 深度查词（词典面板）

**作为**一个想深入了解某个词的用法的用户，
**我想要**点击小卡片后看到更详细的词典信息，
**以便**了解音标、多义项、词形变化和例句。

### 场景

1. 用户点击复习词 → 弹出第一级小卡片
2. 用户点击小卡片 → 顶部展开词典面板（约屏幕 1/4 高度）
3. 面板内容：
   - 词（加粗，较大字号）
   - 音标 `/ˈepɪsəʊd/`
   - 多义项（编号列出）
   - 词形变化（如 episodes）
   - 本集例句 + 历史出现例句（如有）
4. 面板内容可上下滑动
5. 点击面板外区域 → 面板关闭
6. 主内容区在面板打开时整体下移，不遮挡当前阅读位置

### 验收要点

- 面板从顶部向下滑入 0.25s ease-out
- 下方内容区整体下移（不是覆盖）
- 面板内可滚动
- 关闭方式：点击面板外
- 不显示统计数据

---

## US-6: 词汇自动学会

**作为**一个想自然积累词汇的用户，
**我想要**系统在我熟练后自动不再标记这个词，
**以便**我已经掌握的词汇不会一直高亮，干扰阅读。

### 场景

1. 目标词 `episode` 首次出现 → 带释义（`is_new: true`）
2. 第二次出现 → 仅加粗（`is_new: false`），用户点击查了一次
3. 第三次出现 → 仅加粗，用户没查
4. 第四次出现 → 仅加粗，用户没查
5. 连续 3 次出现用户均未点击查询
6. 系统将该词标记为"已学会"
7. 此后该词出现时不再加粗，与普通文字无异

### 验收要点

- M 值默认为 3，后续可配置
- 判定在客户端完成（本地记录），后续同步至服务端
- "已学会"状态可被用户手动撤销（长按已学会的词 → 标记为未掌握）

---

## US-7: 集末词汇面板

**作为**一个读完一集的用户，
**我想要**看到本集我遇到了哪些词，
**以便**获得"我做到了"的正反馈，而非被考核。

### 场景

1. 本集最后一条消息出现后
2. 半屏面板从底部滑入
3. 面板展示：
   - "本集读完"（温灰小字）
   - "遇见 X 个词"（正文，不计数不考核）
   - 词汇列表：新词显示词 + 释义，复习词仅显示词
   - 累积统计（可选）："已读 X 集 · 累积遇见 Y 个词"
   - "继续阅读 →" 按钮
4. 用户看过后点击"继续阅读"进入下一集
5. 如果是最后一集（已追上进度），显示"敬请期待下一集"

### 验收要点

- 措辞：用"遇见"而非"学习"，"本集"而非"任务"
- 不需要用户点击每个词、不需要答题
- 按钮为文字 + 箭头，不用实心按钮
- 面板背景 `#F5F2EB`，与主界面有微弱区分

---

## US-8: 番外集

**作为**一个学习者，
**我想要**在主线剧情无法覆盖某些目标词时，通过番外故事来接触它们，
**以便**所有目标词都有机会在故事中自然遇到。

### 场景

1. 用户正常阅读主线剧集
2. 每 N 集主线后，插入一集番外（`kind: "side"`）
3. 番外在阅读体验上与主线完全相同（同样的聊天界面、同样的点击阅读方式）
4. 番外集末的面板同样展示本集词汇
5. 用户可能不会明显感知到"这是一集番外"——它就是一个有独立小故事的分集

### 验收要点

- 番外集的 JSON 格式与主线相同，仅 `meta.kind` 为 `"side"`
- 番外的故事独立完整，但角色和氛围与前文一致
- N 值可配置（初始默认值待定）
- 阅读界面上不刻意区分主线和番外

---

## US-9: 上传自己的小说和词表

**作为**一个对某部小说有情感连接的用户，
**我想要**上传我喜欢的小说和我想学的词表，
**以便**读我想读的故事，学我想学的词。

### 场景

1. 用户在书架页点击 "+" 按钮
2. 如果尚未配置 API → 提示"请先配置 API 密钥"，提供跳转至 API 设置页
3. 进入上传页
4. 选择上传小说文件（文本文件或直接粘贴文字）
5. 填写作品名（若不填则从文件名自动提取）
6. 上传词表文件（.txt，每行一个单词）
7. 点击"开始生成"
8. 系统调用用户配置的 API 进行内容生成
9. 生成完成后，作品出现在书架上，用户可开始阅读

### 验收要点

- 支持的输入格式：.txt 纯文本、直接粘贴文字
- 词表格式：每行一个单词的 .txt 文件
- 生成期间不要求用户等待（后台处理，通知完成）
- 生成失败时有明确提示和重试方式
- 用户可删除自己上传的作品

---

## US-10: 书架管理

**作为**一个在读多部作品的用户，
**我想要**在书架上看到所有作品的阅读进度，
**以便**在不同作品之间切换阅读。

### 场景

1. 用户从阅读界面返回书架
2. 书架以列表或卡片形式展示所有作品
3. 每部作品显示：封面/标题、已读集数/总集数、最近阅读时间
4. 点击某部作品 → 进入该作品的阅读界面，从上次中断处继续
5. 书架为空时（无内置作品、用户也未上传）→ 显示引导："选择一部作品开始阅读" + "上传你的小说"

### 验收要点

- 书架是 App 的首页（或在阅读界面顶部可返回）
- 进度展示为"Ep.3 / 12"，不显示百分比
- 支持从书架删除作品（确认操作）

---

## US-11: 接续阅读

**作为**一个隔了几天重新打开 App 的用户，
**我想要**直接从上次读到的地方继续，
**以便**不需要记住上次读到哪里、不需要导航。

### 场景

1. 用户关闭 App 时正在阅读 Ep.3 第 15 条消息
2. 几天后用户重新打开 App
3. 直接进入上次的阅读位置，所有历史消息可见
4. 点击继续下一条

### 验收要点

- 阅读位置持久化（本地存储 + 必要时同步）
- 重新打开 App 无需加载、无闪屏
- 如果上次读的作品已被删除，回退到书架

---

## US-12: 切换阅读模式

**作为**一个遇到不适合对话体内容的用户，
**我想要**在需要时切换到常规段落阅读模式，
**以便**那些不适合对话体表达的内容也能顺畅阅读。

### 场景

1. 用户在阅读界面点击设置（角落入口）
2. 看到"阅读模式"选项：对话体 / 段落
3. 切换到段落模式 → 本集以传统段落形式重新渲染
4. 段落模式下词汇标记规则不变
5. 可随时切回对话体

### 验收要点

- 切换模式时不丢失阅读位置
- 段落模式为备选，默认使用对话体
- 设置入口低调，不干扰阅读

---

## US-13: 设置

**作为**一个对阅读体验有个性化偏好的用户，
**我想要**调整字体大小和阅读模式，
**以便**找到最舒适的阅读状态。

### 场景

1. 用户从阅读界面角落进入设置
2. 可调整项：
   - 字体大小（小/中/大）
   - 阅读模式（对话体/段落）
   - API 设置入口（跳转至 API 设置页）
   - 关于 / 反馈入口
3. 设置即时生效，不需要保存按钮

### 验收要点

- 设置项不超过 5 个，保持简洁
- 不包含账号、隐私、通知等复杂项（MVP 阶段）
- 字体大小影响对话和叙述文字

---

## US-14: 配置 API

**作为**一个想用自己的 AI 服务来生成故事的用户，
**我想要**在 App 中配置 API 地址和密钥，
**以便**使用我自己的 API 额度来生成对话体故事。

### 场景

1. 用户从书架或设置页进入 API 设置页
2. 看到以下配置项：
   - **API 地址**：输入框，默认预填 `https://api.deepseek.com`（可修改）
   - **API Key**：安全输入框，输入时显示为密文
   - **模型名称**：输入框，默认预填 `deepseek-v4-pro`（可修改）
   - **测试连接** 按钮
3. 用户填入自己的 Key 后点击"测试连接"
4. 发送一条简短测试请求 → 成功则显示"连接成功 ✓"，失败则显示具体错误原因
5. 配置自动保存在本地安全存储，无需手动保存
6. 用户返回书架，配置生效

### 验收要点

- API Key 存储在设备本地安全区域（Keychain / EncryptedSharedPreferences），不离开设备
- 仅向用户指定的 API 地址发送请求
- App 自身不收集、不上传用户的 API Key
- 默认支持 OpenAI 兼容接口（覆盖面最广）
- 上传作品前，如果用户未配置 API → 在上传页提示"请先配置 API 密钥"
- 测试连接失败时给出可读的提示（网络错误 / 认证失败 / 超时 分别提示）

# 待定决策项

以下是在 PRD 完成前需要确认的关键决策。

## 1. 词汇行为参数

### M 值 —— 连续多少次不查询视为已学会

**已定：M = 3**（后续可配置）。影响：
- 一个词连续 3 次出现且用户均未点击查询 → 标记为"已学会"
- 内容生成时目标词的复现安排策略

### 词形变化匹配规则

**已定：Lemma 归一化。**

- **方案**：Lemmatization — 将词的屈折形式还原为词典原形（consume/consumed/consuming → consume, go/went/gone → go）
- **执行时机**：LLM 生成阶段。生成 prompt 中要求对目标词做 lemma 归一化，确保同一词的不同屈折形式被识别为同一学习对象
- **客户端行为**：按原词（surface form）追踪和展示。`marks` 中 `word` 字段记录的是消息中实际出现的词形（如 `consuming`），但学习追踪按 lemma 合并（`consuming` 和 `consumed` 都归入 `consume` 的学习记录）
- **词典查询**：对变体形式（过去式、复数等）查询词典时，先还原为 lemma 再查询
- **具体引擎**：ECDICT `exchange` 字段。ECDICT 收录了 53K 条带词形映射的条目，格式为 `exchange: "0:go/1:p"`（`0:` 前缀 = lemma）。客户端直接解析此字段即可从屈折形式还原到原形，无需引入外部 lemmatizer 库。已在 `ecdict_mobile.db` 中可用。
- **查询流程**：用户点击词 → 查 ECDICT → exchange 中有 `0:xxx` → lemma = xxx，用该词查完整释义；exchange 为空 → 原词已是原形，直接用。不在 ECDICT 中的词返回"暂无释义"。

## 2. 番外策略参数（MVP 阶段可延后）

### N 值 —— 每多少集主线插入一集番外

当前文档中为占位符"N"。

**建议范围**：3 ~ 5 集。

### "难以嵌入"的判定规则

候选规则：
- 经过 N 集主线的生成尝试后仍未覆盖的词
- 预分类：生僻名词、专业术语优先进入番外列表
- 由 LLM 在生成每集时自行判断并报告未覆盖词

## 3. 客户端平台

**已定：React Native / Expo**

- iOS + Android 双端覆盖
- 一套 TypeScript 代码
- 本地存储：expo-sqlite（词典、作品、阅读进度）+ expo-secure-store（API Key）
- 动画：Reanimated + Gesture Handler
- 后台生成任务：expo-background-fetch

## 4. 成功指标

当前 PRD 缺失。核心候选指标：

| 指标 | 说明 |
|------|------|
| **阅读完成率** | 开始阅读的用户中，读完第一集的比例 |
| **阅读持续天数** | 30 天内用户活跃阅读的天数 |
| **词汇接触数/天** | 用户平均每天遇到的目标词次数 |
| **DAU** | 日活跃用户数 |

需要确定北极星指标以及 MVP 阶段的成功基线。

## 5. MVP 范围边界

| 功能 | 状态 | 理由 |
|------|------|------|
| 内置作品 | MVP 必做 | 无内置作品则用户需要先上传才能体验 |
| API 设置 | MVP 必做 | 用户上传功能的前置依赖 |
| 用户上传小说 | MVP 必做 | 核心差异化价值 |
| 对话体阅读 | MVP 必做 | 核心交互 |
| 逐句点击 | MVP 必做 | 核心交互 |
| 词汇首次带释义 | MVP 必做 | 核心学习闭环 |
| 词汇复习+点击查词 | MVP 必做 | 核心学习闭环 |
| 直接查词（点击目标词打开词典面板） | MVP 必做 | 核心交互 |
| 内置离线词典 | MVP 必做 | 查词不依赖网络 |
| 集末词汇面板 | MVP 必做 | 原则3"读完即正反馈"的载体 |
| 自动学会检测 | MVP 必做 | 原则5 核心闭环 |
| 设置（字体大小等） | MVP 必做 | 实现简单 |
| 番外集 | 可延后 | 依赖词表覆盖度的实际数据 |
| 段落阅读模式 | 可延后 | 备选功能，非核心路径 |

## 已确定的架构决策

以下决策已在讨论中确定，此处仅记录结论：

| 决策 | 结论 |
|------|------|
| 服务端 | 无。App 完全在用户设备上运行 |
| 内容生成 | 用户提供自己的 API Key，App 内置 prompt 和生成流程 |
| 词典数据 | 内置离线 ECDICT 与 WordSenseDB 移动版 SQLite |
| 内置作品 | 预生成 JSON 打包进 App |
| API 兼容 | MVP 支持 OpenAI 兼容接口 |
| 账号系统 | MVP 不做，纯本地存储 |
| API Key 存储 | 设备本地安全区域（Keychain / EncryptedSharedPreferences） |
| 阅读体验 | 完全离线，不依赖网络 |
| 生成时机 | 上传后批量生成（预处理），阅读时直接取 JSON |
| 客户端平台 | React Native / Expo；当前交付目标为 Android |
| 本地存储 | expo-sqlite（结构化数据）+ expo-secure-store（API Key） |
| 词汇追踪模型 | 按 (word, definition) 对追踪，同词异义独立学习 |
| 分词定位 | marks 数组挂载消息上，word index（0-based 字索引）定位 |
| 生成质量校验 | 内置 JSON Schema 校验 + 格式错误自动重试 |
